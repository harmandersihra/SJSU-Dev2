◆ operation

Operation sjsu::I2c::Transaction_t::operation = Operation::kWrite

Defines the starting operation of this transaction. The use of the word "starting", refers to the fact that, the operation can change from Read -> Write if a WriteThenRead() function was called on this structure. In that case, the transaction will start from a Write and then transition to a Read after the necessary number of bytes have been written to the bus. Read operations never transition to Write.

◆ repeated

bool sjsu::I2c::Transaction_t::repeated = false

This flag determins if a "repeat start" condition should be emitted on the bus. This flag being set will transition the hardware from Write to Read mode. If this flag is true, data_out and data_in must be set to an actual buffer (cannot be nullptr).
